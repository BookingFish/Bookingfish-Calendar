# ── Security Practices ──────────────────────────────────────────────
#
# Transport:
#   - All connections secured with TLS 1.2 or higher
#   - HTTPS enforced on all endpoints (HSTS)
#
# Authentication:
#   - Vendor passwords are NEVER stored — one-time auth exchange only
#   - Only expiring bearer tokens (30-day TTL) are stored, encrypted at rest
#   - Tokens are revocable at any time from the vendor dashboard
#
# Email Security:
#   - All outbound mail from @bookingfish.ca uses DKIM signing
#   - SPF record published and enforced
#   - DMARC policy active (reject unauthorized senders)
#
# Payments:
#   - Stripe Connect Express — BookingFish has ZERO access to vendor bank accounts
#   - All card data handled by Stripe (PCI DSS compliant)
#   - Funds go directly to vendor Stripe accounts
#
# Data:
#   - Hosted in Canada
#   - Compliant with LPRPDE (federal) and Loi 25 (Quebec)
#   - Vendor data fully isolated — no cross-vendor data exposure
#   - Client personal information never sold or shared with third parties
#
# Incident Response:
#   - Security vulnerabilities responded to within 48 hours
#   - Data breach notification within 72 hours to affected parties and CAI (Quebec)
#
# ── Reporting a Vulnerability ────────────────────────────────────────
#
# Please report security issues to: support@bookingfish.ca
# Include: description, steps to reproduce, potential impact
# We will acknowledge receipt within 48 hours.
#
